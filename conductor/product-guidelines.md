# Product Guidelines

## Core Principles
- **Performance First:** The engine must prioritize high responsiveness and minimal latency to ensure a tight, arcade-accurate game loop.
- **Accessibility and Clarity:** Visual elements and layouts must remain legible across various terminal sizes and background themes.
- **Interactive Polish:** Beyond static ASCII, the game should employ visual effects like subtle screen-shake on impact, flicker for explosions, and smooth frame-based animations.

## User Experience (UX)
- **Controls:** Support intuitive WASD-style controls (A/D for lateral movement, Space for primary fire) for standard keyboard layouts.
- **Visual Hierarchy:** Game state info (score, lives, waves) must be clearly separated from the playfield.
- **UI Flow:** Provide a seamless, informative transition between menus, gameplay, and the high-score summary.

## Tone and Style
- **Voice and Tone:** Maintain an informative and precise prose style. All game messages, menu labels, and instructions should be clear and functional.
- **Visual Aesthetic:** Strictly follow a retro-digital aesthetic using high-contrast ASCII art and consistent spacing.
